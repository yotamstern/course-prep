FHS:

The difference between /etc/reslov.conf on WSL and on a k8s cluster.


controlplane /etc ➜  cat  ./resolv.conf 
search 5cpiv7fgyty5zt24.svc.cluster.local svc.cluster.local cluster.local
nameserver 10.96.0.10
options ndots:5


user@DESKTOP-SBHAJN3:~$ cat /etc/resolv.conf
nameserver 10.255.255.254
search home


This file in general acts as the config file for the DNS resolver. In regular Linux, it points to the proxy of the windows host that acts as the DNS server. The home is the domain in which the proxy search in for the DNS resolution. In k8s, the name server is probably the CoreDNS which acts as the DNS resolver in the control plane. Since k8s is more focused on cluster forwarding and less on external traffic forwarding, it configed ndots:5 which means that it prioritzes internal cluster routing first.
